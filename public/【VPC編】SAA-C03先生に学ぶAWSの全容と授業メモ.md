## VPCというAWSの基本構造 🌐

AWSを触っていると、かなり早い段階でVPCが出てくる。  
EC2もRDSも、結局はこの中で動く。

最初は意識しなくても進められるけど、  
学習を進めると「VPCを理解してないと厳しいな」と感じる場面が増えてくる。

今回はSAA-C03の内容をベースに、  
VPCまわりを一通り整理する。

## VPCの基本

VPC（Virtual Private Cloud）は、AWS上に作る仮想ネットワーク。

- リージョン単位で作成
- クラウド内外と通信可能
- オンプレミスとも接続できる

VPCの作成数は1リージョンあたり最大5（増加申請可）。  
CIDRレンジは `/16〜/28` の範囲で指定する。

ここでの設計は後から効いてくるので、  
適当に決めると地味に困る。

## デフォルトVPCについて

AWSアカウントを作ると、  
各リージョンにデフォルトVPCが自動で作成される。

同時に以下も設定されている。

- セキュリティグループ
- ネットワークACL
- DHCPオプションセット
- パブリック／プライベートDNSホスト名

とりあえずEC2を立てるには十分。  
ただし、実運用では使わないことも多い。

## VPCウィザード 🧭

VPCを手動で構成すると設定項目が多い。  
VPCウィザードを使えば、

- サブネット
- ルートテーブル
- NATゲートウェイ

などをまとめて作成できる。  
学習段階ではかなり助かる。

## サブネットの考え方

サブネットはVPCを分割したネットワーク。

- サブネットは1AZに属する
- AZをまたぐことはできない
- IPレンジを分けて使う

通信できない原因が、  
だいたいここにあることも多い。

## 外部接続まわり 🚪

VPCを外とつなぐためのコンポーネント。

- インターネットゲートウェイ  
  インターネットとの出入口
- NATゲートウェイ  
  外からの通信を防ぎつつ外へ出る
- Egress Only インターネットゲートウェイ  
  IPv6専用、アウトバウンドのみ
- キャリアゲートウェイ  
  Wavelength（5G）用

用途ごとに役割が分かれている。

## セキュリティ系の整理 🔒

名前が似ていて混乱しやすい。

- ネットワークACL  
  サブネット単位の通信制御
- セキュリティグループ  
  インスタンス単位の通信制御
- DNSファイアウォール  
  DNSクエリの制御
- ネットワークファイアウォール  
  専用サブネットで通信を検査

どこで止めたいか、で考える。

## VPCエンドポイント 🔗

S3やDynamoDBはVPCの外にあるサービス。  
VPCから安全に接続するにはエンドポイントを使う。

- インターネットを経由しない
- 通信がAWS内で完結

S3アクセスで出てきたら、まずこれを思い出す。

## オンプレミス接続

オンプレ接続で出てくる要素。

- カスタマーゲートウェイ  
  オンプレ側
- 仮想プライベートゲートウェイ  
  VPC側
- Site-to-Site VPN  
  拠点間接続
- クライアントVPN  
  個人端末から接続
- Transit Gateway  
  複数VPCやオンプレを集約

規模が大きくなるほどTransit Gatewayの重要度が上がる。

## 管理・可視化系 👀

ネットワークを把握するためのサービス。

- Network Manager
- Reachability Analyzer
- Cloud WAN
- VPC IP Address Manager
- トラフィックミラーリング

障害調査やガバナンス向け。

## 通信の流れ（例）

インバウンド  
インターネット → IGW → 踏み台 → プライベートサブネット

アウトバウンド  
プライベートサブネット → NATゲートウェイ → IGW → インターネット

直接触らせない構成が基本。

## まとめ 📝

- VPCはAWSの土台
- サブネットで分割
- ゲートウェイで外部接続
- FW系で制御ポイントを決める
- エンドポイントで安全にサービス接続

VPCを押さえると、  
AWS全体の構造がかなり見えやすくなる。
